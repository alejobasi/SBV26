## Update Instructions

### After `/onex.specify`
```markdown
- Add new feature to "Features" section
- Update "Total Features" count
- Set status to "Planning"
```

### After `/onex.plan`
```markdown
- Update feature progress: check "Plan created"
- Add timeline dates
- Update links section
```

### After `/onex.tasks`
```markdown
- Update feature progress: check "Tasks defined"
- Add task count
- Update "Total Tasks" in summary
```

### After `/onex.analyze`
```markdown
- Add any issues found to feature "Issues" section
- Update quality metrics if analysis included checks
```

### After `/onex.implement`
```markdown
- Update feature progress: check implementation steps
- Update 🔄 Task Implementation Log table (MANDATORY after each task)
- Update test coverage
- Update quality metrics (SonarQube, security scans)
- Update "Completed Tasks" count
```

### After `/onex.checklist`
```markdown
- Document checklist results in feature section
- Update quality metrics based on findings
- Add any action items to technical debt
```

### After `/onex.taskstoissues`
```markdown
- Add Jira issues mapping link to feature (link to jira-issues-mapping.md)
- Add JQL query for feature issues
- Track Jira issue completion
```
